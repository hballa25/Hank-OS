# Starter Prompt 2 — Stand on proven shoulders

```
before building, study these four open source memory projects: karpathy's llm wiki
(gist.github.com/karpathy/442a6bf555914893e9891c11519de94f), qmd by tobi lutke (github.com/tobi/qmd),
garry tan's gbrain (github.com/garrytan/gbrain), and graphify (github.com/safishamsi/graphify). for
each one tell me what idea is worth stealing for MY setup and what to skip - with reasons. dont copy
any of them wholesale. then update the build plan. ask me 3 questions if anything about my workspace
is unclear.
```

## Hank OS adaptation

Likely verdicts for this vault: Karpathy's "one small index read first" fills Hank OS's biggest gap; Graphify's graph-of-links is already covered by wikilinks + the planned 3D dashboard; gbrain's "answers cite sources / brain cleans itself" maps to the Gardener agent's filing job; qmd's semantic search is a later nice-to-have (keyword + index goes far in a typed vault).
