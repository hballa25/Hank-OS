# Hank OS Supertag Schema

Every typed note carries YAML frontmatter with `type` plus that type's fields. Structure is **retroactive** — captures start untyped in `00 Inbox/`; the Gardener applies types later. Fields may be empty; wikilinks (`[[...]]`) are used for all relations so the graph can see them.

## Core types

### `lesson`
```yaml
type: lesson
unit: "[[Unit Name]]"
standards: ["[[CCSS.XXX]]"]
date: 
resources: []        # wikilinks to assets/Drive/Canva
reflection: ""       # filled after teaching — powers next-year reuse
```

### `unit`
```yaml
type: unit
subject: 
grade: 
standards: []
lessons: []          # backlinked automatically
status: planning | active | done
```

### `standard`
```yaml
type: standard
code: 
subject: 
grade: 
```

### `student`  ⚠️ LOCAL ONLY — never in cloud AI calls
```yaml
type: student
period: 
notes: []
```

### `idea`
```yaml
type: idea
domain: teaching | product | app | life
status: spark | exploring | committed | dropped
```

### `product`  (AI Workbooks etc.)
```yaml
type: product
hustle: "[[AI Workbooks]]"
status: idea | building | ready | launched
assets: []           # Canva designs, files
listing: 
```

### `asset`
```yaml
type: asset
kind: canva | drive | file | image
url: 
used-in: []
```

### `app`  (TaiGrader, XPScholar)
```yaml
type: app
repo: 
status: 
```

### `issue`
```yaml
type: issue
app: "[[TaiGrader]]"
github:              # issue URL
status: open | building | shipped
```

### `trading-plan`  (daily, generated by The Trader)
```yaml
type: trading-plan
date: 
watchlist: []        # tickers/futures
bias: 
key-levels: []
catalysts: []
```

### `trade-journal`
```yaml
type: trade-journal
date: 
plan: "[[YYYY-MM-DD Trading Plan]]"
outcome: 
lesson-learned: ""
```

### `meal-plan`  (weekly, generated by The Chef)
```yaml
type: meal-plan
week-of: 
goal: 
grocery-gaps: []
```

### General
`contact`, `meeting`, `followup` — minimal frontmatter (`type` + `date` + links).

## Conventions
- Filenames are the note title (no dates prefixed except daily notes: `YYYY-MM-DD ...`)
- Tags in body text (`#followup`) are fine as lightweight markers; frontmatter `type` is the real schema
- One note = one thing. Link, don't nest.
